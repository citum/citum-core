Schemas

JSON Schemas

Machine-readable schemas for every Citum data format. Use them in your editor for autocomplete and real-time validation, or in tooling to validate inputs at build time.

Schemas are published automatically by CI from cargo run --bin citum --features schema -- schema --out-dir docs/schemas.

Style authoring

Style

Core Citum style definition: templates, options, citation modes, and bibliography configuration.

style.json

Locale

Terms, date patterns, role labels, and MessageFormat 2 messages for a given language tag.

locale.json

Configuration

Registry

Style registry and alias map. Defines named style entries and their resolution rules.

registry.json

Abbreviation map

Flat key-value map for substituting journal titles, institutional names, and other strings.

abbrev-map.json

Reference and citation data

Bibliography

Input reference format: typed fields, contributors, dates, identifiers, and archive data.

bib.json

Citations

Citation list format with locators, visibility modifiers, and infix text.

citation.json

Server API

Server RPC

JSON-RPC method parameter schemas: render_citation, render_bibliography, validate_style, format_document.

server.json

Editor integration

Add this comment as the first line of any YAML style file to get real-time validation and autocomplete in editors with YAML language server support (VS Code, Helix, Zed, and others).

yaml-language-server hint
# yaml-language-server: $schema=https://docs.citum.org/schemas/style.json

Replace style.json with the appropriate schema filename for your file type.
