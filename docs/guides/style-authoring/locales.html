Locale Messages

Use MessageFormat 2 messages for locale-owned text: labels, role phrases, and compositional patterns.

The messages block

Locale files own language. Style templates express structure; locale files supply the words, inflections, and phrase order via a top-level messages block. Enable the message system with two header fields:

locale-schema-version: "2"
locale: en-US
evaluation:
  message-syntax: mf2
messages:
  term.and: "and"
  term.et-al: "et al."

Message ID namespaces

All message IDs follow a dot-namespaced convention:

  • term.* — localized labels and abbreviations (page markers, connectors)
  • role.* — contributor role phrases, both label and verb forms
  • pattern.* — compositional phrase templates; the locale controls word order and glue

Plain strings and variable substitution

Most messages are plain strings. Use {$variable} to splice in a rendered value supplied by the engine.

messages:
  term.and: "and"
  term.et-al: "et al."
  pattern.accessed-date: "accessed {$date}"
  pattern.in-container: "in {$container}"
  pattern.retrieved-from: "retrieved from {$url}"

Plural dispatch

Use .match {$count :plural} to select between singular and plural forms. when one matches a count of one; when * is the catch-all.

term.page-label: |
  .match {$count :plural}
  when one {p.}
  when * {pp.}

term.chapter-label: |
  .match {$count :plural}
  when one {chap.}
  when * {chaps.}

Gender-aware role labels

Use .match {$gender :select} for gender agreement. Combine with {$count :plural} when both dimensions matter. The $gender argument comes from contributor gender metadata on the reference; when * is the neutral fallback.

# verb form — gender only
role.editor.verb: |
  .match {$gender :select}
  when masculine {editado por}
  when feminine {editada por}
  when * {editado por}

# role label — gender × count
role.editor.label-long: |
  .match {$gender :select} {$count :plural}
  when masculine one {editor}
  when masculine * {editores}
  when feminine one {editora}
  when feminine * {editoras}
  when * * {equipo editorial}

A gender: field on the contributor entry feeds the $gender selector. Mixed-gender groups receive the when * form; Citum does not silently choose a masculine-specific label for a mixed group.

Compositional patterns

pattern.* messages assemble pre-rendered values into natural-language phrases. The engine fills the named variables; the locale controls word order and connecting words. This is where languages differ most — date component order, prepositions, postpositions.

# English — month first
pattern.date-full: "{$month} {$day}, {$year}"

# Spanish — day first
pattern.date-full: "{$day} de {$month} de {$year}"

# Page range
pattern.page-range: "{$start}–{$end}"
ℹ️
Current plural scope MF2 select dispatch (:select) is fully supported, including gendered verb forms. Plural dispatch (:plural) currently recognises one and * (catch-all) — sufficient for most Western-European languages. Full CLDR plural categories (zero, two, few, many) are planned pending ICU4X stabilisation.
