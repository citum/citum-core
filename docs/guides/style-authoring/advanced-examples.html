Advanced Citations

Styles use explicit integral and non-integral branches, per-mode contributor options, and document-scoped integral-name memory where the style opts in. See Templates → Citation modes for the full configuration reference.

Integral vs non-integral (APA 7th)

Style YAML (excerpt)
citation:
  non-integral:
    wrap: parentheses
    template:
      - contributor: author
        form: short
      - date: issued
        form: year
  integral:
    options:
      contributors:
        and: text
Rendered output
Non-integral: (Kuhn, 1962)
Integral:     Kuhn and Popper (1962)

Integral name memory (MLA default)

When integral-names is configured, the first narrative citation uses the full name form; subsequent citations in the same scope use the short form. The scope resets at a chapter boundary.

Style YAML
options:
  integral-names:
    rule: full-then-short
    scope: document
    contexts: body-and-notes
    subsequent-form: short

citation:
  integral:
    template:
      - contributor: author
        form: long
      - variable: locator
        show-label: false
        wrap: parentheses
Rendered output
First mention:      John Smith (10)
Later same scope:   Smith (12)
After chapter reset: John Smith (14)

Document override from front matter

Document front matter can override style-level integral-name settings. Useful when a document requires tighter name tracking than the base style provides.

Djot document front matter
---
integral-names:
  enabled: true
  scope: chapter
  contexts: body-and-notes
---

First narrative mention: [+@smith2010, p. 10]
Later in same chapter:   [+@smith2010, p. 12]
New chapter reset:       [+@smith2010, p. 14]

Archival references and eprints

Citum accepts structured archive-info for manuscripts and related records, and a structured eprint object for preprint-style identifiers. Legacy flat archive and archive_location fields still load for compatibility, but new inputs should use archive-info.

Reference YAML
references:
  - id: dead-sea-scrolls-demo
    class: monograph
    type: manuscript
    title: "The Community Rule (1QS)"
    issued: "-0099"
    genre: "Manuscript scroll"
    archive-info:
      name: "Israel Antiquities Authority"
      location: "Shrine of the Book"
      place: "Jerusalem"

  - id: archive-aware-preprint
    class: monograph
    type: preprint
    title: "Archive-Aware Preprint"
    issued: "2026-02"
    url: "https://arxiv.org/abs/2602.01234"
    archive-info:
      name: "Houghton Library"
      collection: "Ada Lovelace Papers"
      collection-id: "MS Am 1280"
      series: "Correspondence"
      box: "12"
      folder: "4"
      item: "7"
      place: "Cambridge, MA"
    eprint:
      id: "2602.01234"
      server: "arxiv"
      class: "cs.DL"
Rendered output (Chicago notes style)
The Community Rule (1QS). Manuscript scroll, 100 BC,
Israel Antiquities Authority, Shrine of the Book, Jerusalem.

Archive-Aware Preprint. February 2026, Houghton Library,
Ada Lovelace Papers, MS Am 1280, Series Correspondence,
Box 12, Folder 4, Item 7, Cambridge, MA. arxiv:2602.01234 [cs.DL]

Historical EDTF years use astronomical numbering: -0099 renders as 100 BC. The eprint.server value is normalized; supported values include arxiv, ssrn, biorxiv, zenodo, and hal.

Try it
citum render refs \
  -b examples/archival-eprints.yaml \
  -s examples/archive-eprint-demo-style.yaml \
  -k archive-aware-preprint

Advanced dates (EDTF)

Citum uses EDTF (Extended Date/Time Format) as its primary date exchange format. EDTF strings preserve semantic nuances — approximate, uncertain, open-ended — that can be localized by the processor according to the style’s locale.

Date intervals (Level 0)
issued: "2023-05/2024-06"   # Closed interval
issued: "2023-05/.."        # Open start
issued: "../2023-05"        # Open end
Uncertain and approximate (Level 1)
issued: "2004?"              # Uncertain year  → 2004?
issued: "2004~"              # Approximate      → ca. 2004
issued: "2004%"              # Both
issued: "2004-06-11?"        # Uncertain day only
Seasons (Level 1)
issued: "2023-21"            # Spring 2023
issued: "2023-22"            # Summer 2023
issued: "2023-23"            # Autumn 2023
issued: "2023-24"            # Winter 2023
Unspecified digits (Level 1)
issued: "199u"               # Some time in the 1990s
issued: "2004-uu-uu"         # Some time in 2004, month/day unknown

Historical years use astronomical numbering: -0099 = 100 BC. Locale determines how uncertainty modifiers (?, ~) are rendered as text (“?”, “ca.”, “circa”, etc.).

Annotated bibliography

Attach reader-authored annotations to any bibliography without creating style variants. Annotations are passed as a separate YAML or JSON file mapping reference IDs to annotation text. The processor appends them after each entry.

Render with annotations
citum render refs \
  -b examples/annotated-bibliography/references.yaml \
  -s apa-7th \
  --annotations examples/annotated-bibliography/annotations.yaml
Annotations file (YAML)
smith2019: >
  A landmark work that fundamentally reshaped our understanding
  of the field. Particularly useful for its comparative methodology.

hawking1988: >
  Accessible and intellectually ambitious. Balances rigour with
  clarity for a general audience.
Rendered output
Smith, J. (2019). Foundations of knowledge. Oxford University Press.

    A landmark work that fundamentally reshaped our understanding of the
    field. Particularly useful for its comparative methodology.

Hawking, S. (1988). A brief history of time. Bantam Books.

    Accessible and intellectually ambitious. Balances rigour with clarity
    for a general audience.

Annotation text is parsed as Djot inline markup by default: _italic_, *bold*, and [text]{.smallcaps} are rendered into the target output format. To opt out, set format: plain in annotation style options.

Annotations are document-scoped overlays intentionally separate from references. The same reference can carry different annotations in a course syllabus, a grant proposal, and a personal reading list — no style variants needed.

Style HTML output with CSS
.citum-annotation {
    font-style: italic;
    padding-left: 1.5rem;
    margin-top: 0.5rem;
    color: #475569;
}

Abbreviation maps

Substitute full rendered strings with abbreviations at document scope. Pass a YAML or JSON map at render time to replace journal titles, institutional names, corporate authors, and other strings. Substitution applies after rendering, so any style works without modification. See Options → Document-level abbreviation maps for full configuration.

Abbreviations file (YAML)
Estates Gazette: EG
"Lloyd's Law Reports": "Lloyd's Rep"
"World Health Organization": WHO
Nature Reviews Neuroscience: Nat Rev Neurosci
Render with abbreviations
citum render refs \
  -b examples/references.json \
  -s ieee \
  --abbreviation-map examples/abbrevs.yaml
Before and after
# Without map:
Smith, J., et al. "Analysis of inheritance."
Nature Reviews Neuroscience, vol. 24, no. 5, pp. 311–326, 2023.

# With map:
Smith, J., et al. "Analysis of inheritance."
Nat Rev Neurosci, vol. 24, no. 5, pp. 311–326, 2023.

Keys are exact, case-sensitive rendered strings. What gets abbreviated: title fields (main title, container title, collection title), variable fields (publisher, archive, series), and contributor literal names. Abbreviations are document-scoped; keep them separate from the style so the same reference abbreviates differently in different contexts.
