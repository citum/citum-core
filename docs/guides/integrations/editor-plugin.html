Integration recipe

Use Citum as a long-lived process from your editor or plugin

Editor and plugin authors usually do not want to spawn a fresh process per keystroke. Run citum-server once, speak newline-delimited JSON-RPC over stdin/stdout, and you get live citation and bibliography formatting at interactive latency. The same surface is the natural target for word-processor add-ins (Word, LibreOffice, Pages) — Citum doesn't ship those plugins itself, but the integration path is open.

1. Install and start the server

cargo install citum-server
citum-server      # reads JSON-RPC from stdin, writes responses to stdout

Use citum-server --http --port 8765 to enable HTTP mode (requires the http feature). For editor plugins, stdin/stdout is the simpler integration; it sidesteps port allocation, CORS, and process discovery.

2. The wire format

One JSON object per line in, one JSON object per line out. The envelope is { "id", "method", "params" }; responses echo the id and carry either result or error. The most useful method for editor integration is format_document, which renders an entire document's citations and bibliography in one round-trip.

Request (one line in actual use; pretty-printed here)
{
  "id": "req-1",
  "method": "format_document",
  "params": {
    "style": { "kind": "id", "value": "apa-7th" },
    "output_format": "html",
    "refs": {
      "smith2010": {
        "id": "smith2010",
        "class": "monograph",
        "type": "book",
        "title": "Nationalism: Theory, Ideology, History",
        "author": [{ "family": "Smith", "given": "Anthony D." }],
        "issued": "2010",
        "publisher": { "name": "Polity" }
      }
    },
    "citations": [
      {
        "id": "cite-1",
        "items": [
          { "id": "smith2010", "locator": { "label": "page", "value": "10" } }
        ]
      }
    ]
  }
}
Response (excerpt)
{
  "id": "req-1",
  "result": {
    "formatted_citations": [
      {
        "id": "cite-1",
        "text": "(<span class=\"citum-citation\" data-ref=\"smith2010\">Smith, <span class=\"citum-issued\">2010</span>, <span class=\"citum-variable\">p. 10</span></span>)",
        "ref_ids": ["smith2010"]
      }
    ],
    "bibliography": { "format": "html", "content": "..." },
    "warnings": []
  }
}

style is a tagged union: {"kind":"id","value":"apa-7th"} for a registry id, {"kind":"path","value":"/abs/style.yaml"} for a local file, {"kind":"yaml","value":"..."} for inline YAML, or {"kind":"uri","value":"https://..."} for a remote URL. Reference objects use the native Citum schema with a class discriminator. To send CSL-JSON, convert first with citum convert refs legacy.json --from csl-json -o refs.yaml and read it back, or use the file-based render command.

3. A working Node client

Spawn citum-server, write one request, read one response. The fixture under guides/integrations/fixtures/ is what produced the response above — you can run this verbatim against a local install. 

// client.mjs
import { spawn } from "node:child_process";
import { createInterface } from "node:readline";
import { readFileSync } from "node:fs";

const request = JSON.parse(
  readFileSync(new URL("./format-document-request.json", import.meta.url))
);

const server = spawn("citum-server", [], { stdio: ["pipe", "pipe", "inherit"] });
const lines = createInterface({ input: server.stdout });

server.stdin.write(JSON.stringify(request) + "\n");

for await (const line of lines) {
  const response = JSON.parse(line);
  console.log(JSON.stringify(response, null, 2));
  server.stdin.end();
  break;
}
node client.mjs

In a real editor plugin you'd keep the server alive for the session and pipeline requests through it. The protocol is newline-delimited, so any framing library that handles that pattern works.

For browser-based editors

If your editor runs in a browser (Google Docs add-on, web-based IDE, custom Electron app), prefer the WASM bridge. The functions render_citation, render_bibliography, validate_style, get_style_metadata, and materialize_style cover the same surface as the JSON-RPC server, but in-process.

Word, LibreOffice, Pages

Citum doesn't currently ship a Word, LibreOffice, or Pages plugin. The integration shape is well-suited to it — JSON-RPC for desktop add-ins, WASM for web-based ones — and we're happy to support anyone building one. Open a thread on GitHub Discussions to coordinate.

Security note

citum-server resolves styles by id, path, or URI through a resolver chain that can issue outbound network requests (registry, HTTP, git). Do not expose it to untrusted clients. For editor integrations, run it as a local child process; for HTTP mode, bind to localhost.
