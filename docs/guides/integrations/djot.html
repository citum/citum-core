Integration recipe

Write citations in Djot — Citum’s native format

Djot is the format Citum grew up with. It is the default input — no -I flag required — and it supports inline markup in reference fields that the Markdown compatibility mode does not.

1. Install

cargo install citum

Pre-built binaries are also available from GitHub releases.

2. Your first Djot document

Citum uses the same bracketed citation syntax as Pandoc, with one difference: the integral (narrative) form uses a + prefix inside the brackets rather than a bare @key before them.

Manuscript (Djot, Citum citation syntax)
# A short paper

Parenthetical: [@kuhn1962].

Integral: [+@smith2010, p. 10] argues the point.

Author suppressed: [-@watson1953, p. 737].

Multi-cite: [@kuhn1962; @watson1953, ch. 2].
Render
citum render doc manuscript.djot \
  -b references.json \
  -s apa-7th \
  -f html \
  -o paper.html

Djot is the default input format. No -I flag needed.

3. Rich inline markup in reference fields

Djot inline markup — _italic_, *bold*, [text](url), [text]{.small-caps} — is understood natively in the title, note, and abstract fields of Citum YAML references. This lets you encode an embedded book title, a linked URL in a note, or typographic emphasis exactly where it belongs in the data. The Markdown compatibility mode does not do this.

The entry below has a journal article whose title contains an embedded book title. The _..._ marks it for italic rendering within the article title.

refs-rich.yaml (Citum-native YAML, excerpt)
  # title contains an embedded book title requiring italic markup
  - id: hoyningen2015
    class: serial-component
    type: article
    title: "Kuhn's _The Structure of Scientific Revolutions_ Revisited"
    author:
      - family: Hoyningen-Huene
        given: Paul
    issued: "2015"
    pages: "1-18"
    container:
      class: serial
      type: academic-journal
      title: Studies in History and Philosophy of Science
      volume: "52"
Render
citum render doc manuscript-rich.djot \
  -b refs-rich.yaml \
  -s chicago-notes-18th \
  -f html \
  -o paper.html

Chicago footnote output renders the title as “Kuhn’s The Structure of Scientific Revolutions Revisited” — the embedded book title is italicised, the rest of the article title is not.

4. Other output formats

# LaTeX
citum render doc manuscript.djot -b references.json -s apa-7th -f latex -o paper.tex

# Typst
citum render doc manuscript.djot -b references.json -s apa-7th -f typst -o paper.typ

# PDF (renders through Typst; requires the typst CLI on PATH)
citum render doc manuscript.djot -b references.json -s apa-7th --pdf -o paper.pdf

Reproduce this recipe

All fixtures live in guides/integrations/fixtures/: manuscript.djot and manuscript-rich.djot (manuscripts), references.json (CSL-JSON bibliography), and refs-rich.yaml (Citum-native YAML with the markup example). Copy them locally and run any of the commands above.

Using Markdown instead? See Format a Markdown or Djot document on the command line. Need to wire this into a build pipeline? See Add citations to your static-site or docs build.
