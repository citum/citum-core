Integration recipe — experimental (citum-labs)

Use Citum from LuaLaTeX

An experimental LuaLaTeX citation workflow backed by the Citum Rust processor. No Biber, no external bibliography tool — two lualatex passes replace them. All Citum output rules (disambiguation, name formatting, et-al) apply identically across all output paths.

Experimental: this integration lives in citum-labs, not citum-core, and currently targets macOS and Linux. APIs and behaviour may change. For a simpler TeX-free PDF path, see the Typst recipe.

How it works

The citum.sty package loads a Lua module (citum.lua) that calls the Citum processor via LuaJIT FFI. Disambiguation and bibliography rendering require two compilation passes, similar to BibTeX or Biber: 

Pass 1: collects citation keys → writes .citum.json cache
Pass 2: renders final citations and bibliography using the cache

If the Rust shared library is not found, the package automatically falls back to JSON-RPC mode (connecting to a running citum-server instance). You can also force RPC mode with the rpc package option.

1. Build the shared library

The LuaJIT FFI binding loads libcitum_processor, built from crates/citum-bindings in citum-core. 

cargo build -p citum-bindings --release
# macOS: target/release/libcitum_processor.dylib
# Linux: target/release/libcitum_processor.so

2. Load the package in your document

Place citum.sty and citum.lua from citum-labs/bindings/latex/ alongside your document, then load the package with your style and bibliography. 

\usepackage[style=apa-7th, bibfile=refs]{citum}

% Non-integral (parenthetical)
\cite[p.~10]{kuhn1962}               % → (Kuhn, 1962, p. 10)

% Integral (narrative)
\textcite{weinberg1971}              % → Weinberg and Freedman (1971)

% Multi-key parenthetical
\cites{kuhn1962, watson1953}         % → (Kuhn, 1962; Watson & Crick, 1953)

% Print bibliography
\printbibliography

3. Compile (two passes)

Point CITUM_LIB_PATH at the full path of the shared library and run lualatex twice. 

export CITUM_LIB_PATH=$(pwd)/target/release/libcitum_processor.dylib
lualatex paper.tex   # pass 1: collects keys
lualatex paper.tex   # pass 2: renders final output

On Linux replace .dylib with .so. Set CITUM_LUA_PATH if citum.lua is not in the same directory as citum.sty.

Reproduce this recipe

A full working document with bibliography and all citation modes is in citum-labs/bindings/latex/ (citum-example.tex, example-refs.yaml). The Lua binding source is in citum-labs/bindings/lua/.

For a TeX-free PDF path, see the Typst recipe. For C FFI usage from other languages, see Developer → C FFI.
