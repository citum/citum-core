Integration recipe

Format a Markdown or Djot document on the command line

You have a manuscript with [@key] citations and a bibliography. You want HTML, LaTeX, Typst, or PDF out. This is the same workflow you might know from Pandoc, with a different engine doing the citation work.

1. Install

cargo install citum

Pre-built binaries are also available from GitHub releases.

2. Render to HTML

Two ingredients: the manuscript and a bibliography. The bibliography can be CSL-JSON, Citum YAML/JSON, BibLaTeX, or RIS — Citum auto-detects from the extension.

Manuscript (Markdown, Pandoc citation syntax)
# A short paper

Parenthetical citation: [@kuhn1962].

Integral (Pandoc-style narrative): @smith2010 [p. 10] argues the point.

Author suppressed: [-@watson1953, p. 737].

Multi-cite: [@kuhn1962; @watson1953, ch. 2].
Render
citum render doc manuscript.md \
  -I markdown \
  -b references.json \
  -s apa-7th \
  -f html \
  -o paper.html

Drop -I markdown if you use Djot (the default). In Djot the integral form is [+@key] instead of bare @key.

3. Other output formats

# LaTeX
citum render doc manuscript.md -I markdown -b references.json -s apa-7th -f latex -o paper.tex

# Typst
citum render doc manuscript.md -I markdown -b references.json -s apa-7th -f typst -o paper.typ

# PDF (renders through Typst; requires the typst CLI on PATH)
citum render doc manuscript.md -I markdown -b references.json -s apa-7th --pdf -o paper.pdf

4. Coming from BibTeX or CSL-JSON

Convert an existing bibliography to Citum's native YAML once, then point at it. citum render reads BibLaTeX and CSL-JSON directly; converting first is only useful if you want to edit the references by hand. 

# BibLaTeX -> Citum YAML
citum convert refs thesis.bib -o refs.yaml

# CSL-JSON array -> Citum YAML
citum convert refs legacy.json --from csl-json -o refs.yaml

Reproduce this recipe

The exact fixture used to verify these commands lives in guides/integrations/fixtures/: manuscript.md, manuscript.djot, and references.json. Copy them locally and run any of the commands above.

Need to wire this into a build pipeline? See Add citations to your static-site or docs build. Need live formatting from an editor or plugin? See Use Citum as a long-lived process.
