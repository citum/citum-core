Integration recipe

Publish to PDF with Typst

Citum renders citations and bibliography entries to Typst markup (-f typst). Pass that output to the Typst CLI to compile to PDF. Citum handles all citation formatting — disambiguation, grouping, note behavior, link resolution — and Typst handles page layout and font rendering.

Direct PDF compilation via --pdf is not available in the current crates.io release (the helper crate is not yet published). The two-step workflow below works with any standard Typst installation.

1. Install both tools

Citum
cargo install citum
Typst CLI
# macOS / Linux via Homebrew
brew install typst

# Or via Cargo
cargo install typst-cli

2. Render to Typst markup

Use -f typst to run Citum’s full citation engine and emit a .typ file. Any Citum-supported input format (.djot, .md) and bibliography format (YAML, CSL-JSON, CBOR) work here. 

citum render doc manuscript.djot \
  -b refs.json \
  -s apa-7th \
  -f typst \
  -o paper.typ

3. Compile to PDF

Pass the generated .typ file to Typst. The resulting PDF includes internal citation-to-bibliography links and live DOI/URL links. 

typst compile paper.typ paper.pdf

Combined in one shell line: 

citum render doc manuscript.djot -b refs.json -s apa-7th -f typst -o paper.typ \
  && typst compile paper.typ paper.pdf

What you get

Full Citum output rules

Citation disambiguation, bibliography grouping, name et-al, and note numbering all apply identically to the HTML and LaTeX paths. Typst is a rendering backend, not a bibliography engine.

Interactive PDF links

Internal citation-to-bibliography jumps and external DOI/URL links are embedded by Typst. Advanced hover behaviors and metadata popups are not currently part of the Typst output path.

Prefer a TeX-native document with \cite{} commands? See the LuaLaTeX integration instead.

Reproduce this recipe

Fixtures in guides/integrations/fixtures/: manuscript.djot (source document) and references.json (CSL-JSON bibliography). Copy them locally and run the commands above.

Rendering to HTML or LaTeX instead? See Write citations in Djot for the full output-format overview. Building in CI or a Makefile? See Add citations to your static-site or docs build.
