Integration recipe

Add citations to your static-site or docs build

You have a directory of Markdown (or Djot) files with [@key] markers and one shared bibliography. You want citations rendered as part of your existing build, not as a separate manual step. This recipe wires citum into a Makefile — the same idea translates to npm scripts, justfiles, or CI jobs.

1. Install citum on the build machine

cargo install citum

2. One source file, one command

The unit is one input document plus one (or more) bibliography files. Output format is controlled by -f; output path by -o. 

citum render doc content/post.md \
  -I markdown \
  -b references.json \
  -s apa-7th \
  -f html \
  -o public/post.html

Pass -b multiple times to merge bibliographies. Use -c citations.yaml if your citations live outside the document.

3. Wire it into a Makefile

A pattern rule keeps one citum invocation per source file. Add a dependency on references.json so any bibliography edit triggers a rebuild. 

STYLE := apa-7th
REFS  := references.json
SRC   := $(wildcard content/*.md)
OUT   := $(patsubst content/%.md,public/%.html,$(SRC))

public/%.html: content/%.md $(REFS)
	@mkdir -p $(@D)
	citum render doc $< -I markdown -b $(REFS) -s $(STYLE) -f html -o $@

build: $(OUT)
.PHONY: build

Other build systems

The shape is the same in every build tool: one shell invocation per document. For npm scripts, wrap the same command in a Node script that iterates the source directory. For CI, run cargo install citum in a setup step and call the Makefile target. For pre-rendering inside a static-site generator that already supports shortcodes or filters, call citum render doc per page and inject the output.

Reproduce this recipe

Fixture: guides/integrations/fixtures/ contains manuscript.md and references.json. Running the literal command above against those files produces well-formed HTML with citations and a bibliography — see the advanced examples for more complex scenarios.
