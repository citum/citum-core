Document authoring

Djot document authoring

Djot is Citum’s native document format. When you write a Djot manuscript and render it with Citum, you get citations, a formatted bibliography, and rich text in reference fields — all from a single command. This page documents what the pipeline supports and how to use it. The live demo shows a complete example rendered with APA 7th.

The document pipeline

A Djot document is a plain text file with citation keys inside brackets. The Citum CLI reads the document, resolves each key against your reference file, and produces a formatted output with inline citations and a bibliography appended at the end.

Minimal Djot document
---
options:
  style: apa-7th
---

As [@kuhn1962] argued, paradigm shifts are rarely smooth.
[+@chen2017] extends this analysis to citation practices.

# Bibliography

::: bibliography
:::
Render to HTML
citum render doc manuscript.djot -b references.yaml -s apa-7th -f html -o paper.html

The ::: bibliography ::: fenced div marks where the bibliography is inserted. Without it, Citum appends the bibliography after the last paragraph. Options can also be supplied via YAML frontmatter in the document itself or via CLI flags — flags override frontmatter.

Citation syntax

Citum uses the same bracketed citation syntax as Pandoc, with one difference: integral (narrative) citations use a + prefix rather than a bare @key before the bracket.

Form Syntax Rendered (APA)
Non-integral (parenthetical) [@kuhn1962] (Kuhn, 1962)
Integral (narrative) [+@chen2017] Chen (2017)
Author suppressed [-@watson1953] (1953)
With locator [@chen2017, ch. 3] (Chen, 2017, ch. 3)
Integral with locator [+@chen2017, p. 45] Chen (2017, p. 45)
Multiple citations [@kuhn1962; @watson1953] (Kuhn, 1962; Watson, 1953)
Locator types. Citum understands standard locator labels: p., pp., ch., §, ¶, fig., no., vol., art., and others. Locators are passed through to the style, which decides how to render them.

Name memory

When the same author appears multiple times as an integral citation, name memory suppresses the given name on repeat occurrences — exactly as a human copy-editor would. This is opt-in and controlled per document.

Document frontmatter
---
options:
  integral-name-memory:
    enabled: true
    scope: document       # "document" or "section"
    contexts: body-and-notes   # or "body-only", "notes-only"
---
Effect in the text (APA author-date)
[+@chen2017] argues …       → Chen (2017) argues …
[+@chen2017, ch. 3] shows … → Chen (2017, ch. 3) shows …   ← given name suppressed
[+@chen2020] extends …     → Chen (2020) extends …          ← still suppressed

The scope controls when the memory resets: document means once the author has appeared, their name is short for the rest of the document. section resets the memory at each top-level heading. The demo shows name memory in action with the Chen citations.

Rich text in reference fields

Djot inline markup is understood natively in the title, note, and abstract fields of Citum YAML references. This lets you encode an embedded book title, a proper noun that should keep its case, or typographic emphasis directly in the data — once, without duplicating it per output format.

Title-within-title

If an article title contains a book title, mark the embedded title with Djot italic (_..._). The renderer applies the appropriate convention per output format: <em> in HTML, \emph{} in LaTeX, #emph[] in Typst. 

- id: hoyningen2015
  class: serial-component
  type: article
  title: "Kuhn's _The Structure of Scientific Revolutions_ Revisited"
  author:
    - family: Hoyningen-Huene
      given: Paul
  issued: "2015"
  container:
    class: serial
    type: academic-journal
    title: Studies in History and Philosophy of Science
    volume: "52"

Chicago output: “Kuhn’s The Structure of Scientific Revolutions Revisited” — the embedded book title is italicised; the rest is not.

Case protection with {.nocase}

Title-case styles would normally uppercase words in a title. Use a {.nocase} span to protect acronyms, proper nouns, or phrases that must stay as written. 

title: "The role of {.nocase}[DNA] in evolutionary theory"
title: "Chronology of the {.nocase}[BCE] period in ancient Rome"

Markup in note and abstract

The note and abstract fields also accept Djot inline markup, including links. This is useful for annotation-style bibliographies. 

- id: smith2024
  type: book
  title: Open Access and the Academy
  author:
    - family: Smith
      given: Jordan
  issued: "2024"
  note: "Preprint available at [arXiv:2401.12345](https://arxiv.org/abs/2401.12345)."

Bibliography grouping

A document can split its bibliography into named sections using fenced divs with filter attributes. Each ::: bibliography ::: div renders only the entries that match its filter; entries that match no filter do not appear.

Two-part bibliography: primary and secondary sources
{ type="manuscript" title="Primary Sources" }
::: bibliography
:::

{ not-type="manuscript" title="Secondary Sources" }
::: bibliography
:::

The title attribute sets the heading for that section. The type attribute filters by Citum entry type; prefix any attribute with not- to invert the match.

Available filter attributes

Attribute Matches
type="book" Entries with that Citum entry type
not-type="book" Entries whose type is not that value
language="ja" Entries with that BCP 47 language tag
title="Section heading" Sets the rendered heading for this bibliography section
The live demo shows primary/secondary bibliography grouping using type="manuscript" to separate an archival source from the secondary literature.

Special reference types

EDTF dates (uncertain and approximate)

Citum reference YAML accepts EDTF Level 1 date strings in the issued field. The two most common modifiers:

Modifier Meaning Example Rendered (APA)
? Uncertain issued: "1891?" (1891?)
~ Approximate issued: "2022~" (ca. 2022)

Archival sources with archive-info

Unpublished manuscripts and archival items use type: manuscript and an archive-info block with repository metadata. 

- id: harrington1891
  class: monograph
  type: manuscript
  title: "Notes on Bibliographic Method and the Organisation of Learned References"
  author:
    - family: Harrington
      given: Robert Langton
  issued: "1891?"
  archive-info:
    name: "British Library"
    location: "Manuscripts Reading Room"
    place: "London"
    collection: "Harrington Papers"
    shelfmark: "MS Add. 47903, ff. 12–34"

Preprints with eprint

Preprints and working papers use type: report with an eprint block that identifies the server and ID. 

- id: okafor2024
  class: monograph
  type: report
  title: "Network Topology of Academic Citation: A Large-Scale Computational Analysis"
  author:
    - family: Okafor
      given: Nnamdi
  issued: "2024-01"
  eprint:
    id: "2401.05832"
    server: arXiv
    class: "cs.DL"

Multilingual sources

Titles and contributor names can carry parallel-script and transliterated forms. The style selects which view to render. See Style authoring and the demo for Japanese and Spanish examples.

Output formats

The same Djot document and reference file can target HTML, LaTeX, or Typst with a single flag change. Rich text markup in reference fields is translated appropriately for each format. 

# HTML
citum render doc manuscript.djot -b references.yaml -s apa-7th -f html -o paper.html

# LaTeX (produces a .tex file for inclusion in your LaTeX project)
citum render doc manuscript.djot -b references.yaml -s apa-7th -f latex -o paper.tex

# Typst
citum render doc manuscript.djot -b references.yaml -s chicago-notes-18th -f typst -o paper.typ

# Compile Typst → PDF (requires the typst CLI)
typst compile paper.typ paper.pdf
Output format and style are independent. Any style can render to any output format. A Chicago-notes style renders footnotes in all three formats; an author-date style renders parenthetical citations in all three.

Try it

The demo page shows a complete Djot manuscript rendered with APA 7th, including name memory, multilingual sources, EDTF dates, archival and preprint entries, and grouped bibliographies. The source files (docs/demo.djot and docs/demo-refs.yaml) are in the repository and can be run locally with: 

citum render doc docs/demo.djot -b docs/demo-refs.yaml -s styles/embedded/apa-7th.yaml -f html
