What Citum is for

CSL remains the compatibility baseline. Citum's goal is to make citation behavior easier to describe: bibliography grouping, multilingual terms, reusable style families, and output that can be checked against real examples.

Handle real bibliography structure

A bibliography is not always one sequence. Citum has room for grouping, sorting, type variants, note behavior, and specialized reference shapes without burying that logic in renderer-specific conventions.

Render across languages and locales

Citation terms are not just English strings. Locale files own role labels, locators, grammatical gender, and the multilingual details that styles should not hardcode.

Reuse styles without hidden inheritance

A dependent style should not become a mystery fork. Base styles, scoped overrides, version requirements, registries, and `extends-pin` make reuse inspectable.

Check the output

A style is only useful if its output can be checked. Citum pairs validation with oracle comparisons, strict fixtures, and CI-generated compatibility reports.

Know what a style depends on

Styles can say which Citum version they need, and the docs can show when support for a capability landed. That matters when styles are shared across journals, registries, and local publishing workflows.
