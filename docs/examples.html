What Citum can do

A quick tour of Citum’s feature set. Each card shows real rendered output and links to the full guide or recipe for that feature.

Citation modes — integral and non-integral

Styles declare separate integral and non-integral templates. Document-scoped name memory tracks whether a name has already appeared in the prose and switches to its short form on subsequent mentions.

Rendered output (APA 7th)
Non-integral:  (Kuhn, 1962, p. 10)
Integral:      Kuhn and Popper (1962)
Name memory:   Thomas S. Kuhn (10) … later Smith (12)

→ Advanced citation examples  ·  Template reference

Bibliography grouping and sorting

Group entries by type, language, or custom criteria. Sort within groups independently. Gender-aware locale terms adapt contributor labels to the language of the source.

Rendered bibliography (grouped by type)
Books
  Kuhn, T. S. (1962). The structure of scientific revolutions. Chicago.
  Watson, J. D. (1968). The double helix. Atheneum.

Articles
  Smith, J. A. (2023). Analysis of inheritance. Nature, 24(5), 311–326.

→ Options → Bibliography sort and groups

Multilingual and script partitioning

References in multiple languages are automatically partitioned by script and language. Locale terms, contributor labels, and date formats adapt per-reference.

Rendered output (MLA, mixed script)
English entry:   Smith, John A. Foundations of Knowledge. Oxford UP, 2019.
Japanese entry:  田中一郎。『科学の構造』。東京大学出版会、2020年。
Spanish entry:   García, María. "La ciencia moderna." Ciencias, vol. 5, 2021.

→ Locales and gender-aware terms

EDTF dates — uncertain, approximate, seasonal

Citum uses EDTF strings as its date exchange format. Semantic nuances are preserved and localized by the processor.

Input → rendered (APA, en-US)
"2004?"        → 2004?
"2004~"        → ca. 2004
"2023-05/.."   → May 2023–
"2023-22"      → Summer 2023
"-0099"        → 100 BC

→ Advanced examples → EDTF dates

Archival references and eprints

Structured archive-info fields model collection hierarchy, shelfmarks, and institutional provenance. A structured eprint object handles preprint-style identifiers (arXiv, SSRN, bioRxiv, Zenodo, HAL).

Rendered output
The Community Rule (1QS). Manuscript scroll, 100 BC,
Israel Antiquities Authority, Shrine of the Book, Jerusalem.

Archive-Aware Preprint. February 2026, Houghton Library,
Ada Lovelace Papers, MS Am 1280. arxiv:2602.01234 [cs.DL]

→ Advanced examples → Archival references

Annotated bibliography

Pass a separate annotations file at render time. Works with any style, no variants needed. Annotation text is parsed as Djot inline markup.

Rendered output
Smith, J. (2019). Foundations of knowledge. Oxford University Press.

    A _landmark_ work. Particularly useful for its comparative methodology.

→ Advanced examples → Annotated bibliography

Pluggable output formats

A single style and bibliography renders to HTML, LaTeX, Typst, Djot, or plain text. The same citation logic applies across all formats.

CLI flags
citum render refs -b refs.json -s apa-7th -f html
citum render refs -b refs.json -s apa-7th -f latex
citum render refs -b refs.json -s apa-7th -f typst
citum render refs -b refs.json -s apa-7th -f plain

→ Djot recipe → Other output formats

PDF publishing

Typst

Render to Typst markup with Citum, compile to PDF with the Typst CLI. Internal citation links and DOI/URL links included.

LuaLaTeX (experimental)

Two-pass LuaLaTeX workflow via LuaJIT FFI or JSON-RPC. No Biber, no external bibliography tool. Lives in citum-labs.

Interactive HTML

Drop two assets into any page with Citum HTML output: citations become clickable, scroll to their bibliography entry, and show hover previews. 

<link rel="stylesheet" href="https://docs.citum.org/assets/citum-interactive.css">
<script src="https://docs.citum.org/assets/citum-interactive.js"></script>

→ Live demo  ·  Developer → Interactive HTML

Binary performance (CBOR)

Styles and bibliographies can be compiled to CBOR for near-instant loading in production environments. 

# Compile style to binary
citum convert style styles/apa-7th.yaml -o apa-7th.cbor

# Use binary inputs at render time
citum render refs -b data.cbor -s apa-7th.cbor
